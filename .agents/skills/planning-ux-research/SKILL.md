# Planning UX Research

## When to Use

- User asks how to validate a design or flow.
- Team needs a practical research plan with low overhead.
- Product decisions require user evidence.

## Workflow

- [ ] Define decision to inform.
- [ ] Select method (interview, usability test, survey, mixed).
- [ ] Define participant profile and sample size.
- [ ] Draft moderator guide and task script.
- [ ] Define synthesis template and decision criteria.

## Instructions

1. Always start with decision-first framing:
   - What decision will this research unlock?
2. Choose method by risk type:
   - Behavior risk -> moderated usability test.
   - Problem-space risk -> discovery interviews.
   - Preference signal -> lightweight survey.
3. Define recruitment criteria:
   - Must-have traits.
   - Exclusion traits.
   - Segment quotas.
4. Draft script with:
   - Warm-up questions.
   - Task scenarios.
   - Follow-up probes.
   - Debrief questions.
5. Define synthesis and decision rule:
   - Affinity notes.
   - Top issues by frequency and severity.
   - Go / Iterate / Stop recommendation.

## Output Format

Save as `results/ux-research-plan.txt`:

DECISION TO INFORM: <decision>
METHOD: <method>
PARTICIPANTS: <profile + n>
RECRUITMENT CRITERIA:

- Include: ...
- Exclude: ...

SESSION PLAN:

- Intro: ...
- Tasks: ...
- Probes: ...

DATA TO CAPTURE:

- ...

SYNTHESIS PLAN:

- ...

DECISION CRITERIA:

- Go: ...
- Iterate: ...
- Stop: ...

## Quality Bar

- Plan maps clearly to one concrete product decision.
- Participant criteria are specific and recruitable.
- Output includes explicit decision thresholds.
